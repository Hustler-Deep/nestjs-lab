# Contributing to NestJS Lab

First off, thanks for taking the time to contribute! ❤️

All types of contributions are welcome, from bug reports to new features and documentation improvements.

## How Can I Contribute?

### Reporting Bugs
- Use the GitHub Issue Tracker.
- Describe the steps to reproduce the bug.
- Mention your environment (Node.js version, OS).

### Suggesting Enhancements
- Open an issue with a clear title and description.
- Explain why this enhancement would be useful for everyone.

### Pull Requests
1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. Ensure the test suite passes.
4. Issue a pull request!

## Style Guide
- Follow the existing NestJS module structure.
- Add comments for complex logic.

## Adding a New Lab Module
If you're adding a new laboratory (e.g., `nestjs-auth0`):
1. Create a new directory at the root: `mkdir nestjs-<name>`.
2. Initialize it with a descriptive `README.md`.
3. Add a summary of the new module to the root `README.md` in the **Project Modules** table.
4. Ensure the new module follows the project's aesthetic and quality standards.

## Questions?
If you have any questions, feel free to open an issue or reach out to the maintainer.

Happy Coding! 🚀
